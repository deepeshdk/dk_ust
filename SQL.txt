                      //SubQuery
->A query inside another query.
->We can access data from different tables.  
->In subquery first inner query will execute and than output of that inner query will give to outer query as input and then outer query will execute
and generate the required resultant set.
syntax:
select * from table_name where column_name operator(select * from table_name);
      ^                                                     ^
      |                                                     |
     Outer Query                                          Inner Query
->Two types of SubQuery:
1.Single Row SubQuery: In these we are using Relational Operator because we are getting single output from inner query.
syntax: select * from table_name where column_name relational operator(select * from table_name);

2.Multi Row SubQuery: Gives more than one resultset so we use Special Operator here.
syntax: select * from table_name where column_name special operator(select * from table_name);

3.Outer Join:
Three Types:
a.Left Outer Join: 
->Join return all the rows of the table on the left side of the join and matching rows of the table on the right side of join.
->It is the combination of Inner join and unmatched records present in the left table.
->The rows for which there is no matching rows on the right-side the result-set will contain null.
synatx: ANSI
select * from table_name left outer join table_name2 on condition;
synatx: Oracle
select * from table_name1,table_name2 where table_name1.column_name=table_name2.column_name(+);

b.Right Outer Join:
->Join return all the rows of the table on the right side of the join and matching rows of the table on the left side of join.
->It is the combination of Inner join and unmatched records present in the right table.
->The rows for which there is no matching rows on the left-side the result-set will contain null.
synatx: ANSI
select * from table_name right outer join table_name2 on condition;
synatx: Oracle
select * from table_name1,table_name2 where table_name1.column_name(+)=table_name2.column_name;

c.Full Outer Join:
->It obtains the unmatched records from both the tables.
synatx: ANSI
select * from table_name1 Full Outer Join table_name2 on condition;


------------------------------------------------------------------------------
4.Natural Join:
->It is similar to inner join where we don't specify the condition.
->It provides inner join if there are common column present else it will give cartisian join.
synatx: ANSI
select * from table1 natural join table2;
